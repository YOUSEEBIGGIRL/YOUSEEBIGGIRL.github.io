剑指 Offer 57 - II. 和为s的连续正数序列

2021-05-04
2分钟阅读时长

输入一个正整数 target ，输出所有和为 target 的连续正整数序列（至少含有两个数）。

序列内的数字由小到大排列，不同序列按照首个数字从小到大排列。

示例 1：

输入：target = 9 输出：[[2,3,4],[4,5]]

示例 2：

输入：target = 15 输出：[[1,2,3,4,5],[4,5,6],[7,8]]

限制：

1 <= target <= 10^5

方法 1 滑动窗口

通过双指针维护一个窗口，并根据窗口中的和值，来进行相应的移动处理，具体为下：

  1. 和值小于所需的值，扩大窗口，操作为右指针 + 1
  2. 和值大于所需的值，缩小窗口，操作为左指针 + 1
  3. 和值等于所需的值，记录结果，左右指针同时 + 1

流程如下图所示：

代码如下：

func findContinuousSequence(target int) [][]int {
    var res [][]int
    // 构建一个数组用于滑动窗口
	n := make([]int, (target>>1)+1)
	for i := 0; i < len(n); i++ {
		n[i] = i + 1
	}

	i, j := 0, 1
	sum := n[i] + n[j]
	for i < j {
		if sum < target {
			j++
			sum += n[j]
		} else if sum > target {
			sum -= n[i]
			i++
		} else if sum == target { 
			var temp []int
			for i1 := i; i1 <= j; i1++ {
				temp = append(temp, n[i1])
			}
			res = append(res, temp)
			sum -= n[i]
			i++
            // j 不是最后一个元素时，才执行下面的操作，
            // 否则 j++ 后会导致 n[j] 越界
			if j < len(n)-1 {
				j++
				sum += n[j]
			}
		}
	}
	return res
}

优化一下

虽然结果是正确的，但是在参考了题解后，发现自己的代码还有一些可以优化的地方，首先是最开始的构建数组，这是完全没必要的，因为是连续正整数序列，所以可以直接把指针作为数字，而不是下标。

此外，对于 sum == target 这种情况，我的处理逻辑是 i 和 j 都移动，但是实际上只需要移动 i 即可，这样可以省去越界判断，且可接减少移动次数。

减少移动次数示例： i 和 j 都移动

  1. i = 4, j = 5, s = 9 == target
  2. i++ = 5, j++ = 6, sum= 11 > 9
  3. i++ = 6，i == j ，break

只移动 i

  1. i = 4, j = 5, s = 9 == target
  2. i =5, j = 5, i == j ，break

代码如下：

func findContinuousSequence(target int) [][]int {
    var res [][]int

	i, j, sum := 1, 2, 3
	for i < j {
		if sum < target {
			j++
			sum += j
		} else if sum > target {
			sum -= i
			i++
		} else if sum == target { 
			var temp []int
			for i1 := i; i1 <= j; i1++ {
				temp = append(temp, i1)
			}
			res = append(res, temp)
			sum -= i
			i++
		}
	}
	return res
}
