使用 Multipass 快速搭建虚拟机

2023-05-19
2分钟阅读时长

之前搭建 k8s 集群一直使用的是 Vmware，虽然总的来说用起来还算 OK，但是总感觉比较重，安装系统什么的也比较耗时（后来发现可以先创建好一台，然后克隆），加上 k8s 老被我搞坏，每次搞坏又都要重新创建集群（貌似可以做一个快照），搞得有点烦，就想看看有没有什么别的虚机工具，能够方便且快速的搭建虚机，最好还能支持命令行操作，找来找去，最终还是找到了之前用过的老朋友 multipass

开始创建

只需要使用一行命令即可快速搭建一台虚机：

multipass launch --name vm

PS：如果是第一次创建需要拉取镜像，会稍微慢一些，之后创建就很快了。

创建时指定配置

此外，你还可以在创建时指定虚机的配置：

multipass launch --name vm --cpus 4 --disk 150G --memory 8G

创建桥接模式的虚机

默认情况下，使用 multipass 创建出来的虚机只能被宿主机访问，同局域网的其他机器无法访问，如果你想创建类似 Vmware 桥接模式的虚机，让同局域网任意机器访问，可以在创建时指定 --network 参数实现。

首先执行 multipass networks 查看一下宿主机的网卡：

multipass networks
Name              Type      Description
Default Switch    switch    Virtual Switch with internal networking
ExtSwitch (WLAN)  switch    Virtual Switch with external networking via "WLAN" (Created by Multipass)
WLAN              wifi      Intel(R) Wi-Fi 6E AX210 160MHz

然后选择一张物理网卡作为 –network 的值，这里我选择 WLAN 这张：

multipass launch --name vm --network WLAN --cpus 4 --disk 150G --memory 8G

然后创建出来的虚拟机会分配一个局域网 IP，如下：

而且这个东西最 NB 的是，宿主机的 clash tun 模式对这些虚机也有效，我之前使用 Vmware 桥接模式搭建的虚机是无效的，不知道怎么做到的

创建时指定配置

我们创建虚机后，往往都需要做一些配置，比如设置 root 用户，配置 ssh 允许以 root 身份登录，安装一些包等等，有点麻烦，更麻烦的是如果要搭建的是集群，那么就要在每台虚拟机上都执行这一系列操作，想想就蛋疼，有没有一种可能，通过配置文件定义好这些配置行为，然后创建时通过指定配置文件的方式，自动执行这一系列行为呢？看了一下官方文档，发现 multipass 还真支持

根据我的要求，我的配置文件定义如下（配置文件的具体参数，可以查阅官方文档）：

#cloud-config
users:
  - name: root
    lock_passwd: false
    plain_text_passwd: "root"
    sudo: ALL=(ALL) NOPASSWD:ALL
    shell: /bin/bash

ssh_pwauth: true

write_files:
  - path: /etc/ssh/sshd_config.d/99-allow-root-login.conf
    content: |
      PermitRootLogin yes

packages:
  - socat
  - conntrack
  - ebtables
  - ipset

/etc/ssh/sshd_config.d/99-allow-root-login.conf 是什么文件？和 /etc/ssh/sshd_config 有什么区别 ?

文件 /etc/ssh/sshd_config.d/99-allow-root-login.conf 是 OpenSSH 服务器配置文件的一个片段文件，它用于存储自定义的配置选项，以覆盖主配置文件 /etc/ssh/sshd_config 中的默认选项。这种分离的配置文件结构使得管理和组织配置更加灵活和可扩展。

主配置文件 /etc/ssh/sshd_config 包含了 OpenSSH 服务器的所有配置选项。它是一个全局配置文件，影响整个服务器的行为。您可以在此文件中设置各种选项，例如监听端口、允许的认证方式、访问控制规则等。

而 /etc/ssh/sshd_config.d/ 目录是用于存放额外的配置文件片段的目录。这些配置文件片段以 .conf 扩展名结尾，并按照一定的顺序加载到主配置文件中。每个片段文件包含一组相关的配置选项，可以根据需要进行添加、修改或删除。这样的分离配置文件的机制使得配置管理更加方便，可以避免直接修改主配置文件，而是通过添加或修改相应的片段文件来实现自定义配置。

例如，/etc/ssh/sshd_config.d/99-allow-root-login.conf 文件可以包含一条配置选项 PermitRootLogin yes，用于允许 root 用户通过 SSH 登录。这样的自定义配置片段文件可以方便地管理和维护，而不必直接修改主配置文件。

然后在创建虚机时，通过 –cloud-init [filepath.yaml] 的方式，指定要应用的配置文件，非常的方便：

multipass launch --cloud-init cloud-config.yaml --name vm --network WLAN --cpus 4 --disk 150G --memory 8G

现在想搭建一个集群就非常容易了，比如下面的命令就可以创建 6 台虚机，组成一个集群：

multipass launch --cloud-init cloud-config.yaml --name kmaster1 --network WLAN --cpus 4 --disk 150G --memory 8G
multipass launch --cloud-init cloud-config.yaml --name kmaster2 --network WLAN --cpus 4 --disk 150G --memory 8G
multipass launch --cloud-init cloud-config.yaml --name kmaster3 --network WLAN --cpus 4 --disk 150G --memory 8G
multipass launch --cloud-init cloud-config.yaml --name kworker1 --network WLAN --cpus 4 --disk 150G --memory 8G
multipass launch --cloud-init cloud-config.yaml --name kworker2 --network WLAN --cpus 4 --disk 150G --memory 8G
multipass launch --cloud-init cloud-config.yaml --name kworker3 --network WLAN --cpus 4 --disk 150G --memory 8G
