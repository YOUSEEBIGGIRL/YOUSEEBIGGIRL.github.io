搭建 NFS StorageClass

搭建 NFS Server

# 安装 NFS 服务端
sudo apt-get install nfs-kernel-server  

# 创建一个 dir 作为 NFS 共享目录
mkdir /share

# 编辑配置
sudo vim /etc/exports
# 在最后一行添加
/share *(rw,sync,no_root_squash,no_subtree_check)

/share *(rw,sync,no_root_squash,no_subtree_check)

其中 /share 就是我们的 NFS 共享目录

*(rw,sync,no_root_squash,no_subtree_check) 是一条配置行，用于指定 NFS 共享目录的权限设置

具体解释如下：

  • *: 表示允许任何客户端访问该共享目录。也可以指定特定的 IP 地址、IP 地址范围或域名等来限制客户端的访问。
  • (rw): 表示共享目录以读写 (read-write) 模式共享，客户端可以对共享目录进行读取和写入操作。也可以使用 (ro) 表示只读 (read-only) 模式共享，客户端只能对共享目录进行读取操作。
  • (sync): 表示使用同步 (synchronous) 模式，NFS 服务器在客户端请求完成之前会等待写入操作同步到磁盘上。这样可以保证数据的一致性，但可能会影响性能。
  • (no_root_squash): 表示允许客户端使用 root 权限访问共享目录。默认情况下，NFS 服务器会将客户端使用 root 权限的请求映射为使用匿名用户 (nobody) 权限，这样可以提高安全性。使用 (no_root_squash) 参数可以禁用这种映射，允许客户端使用 root 权限访问共享目录。
  • (no_subtree_check): 表示禁用子树检查，这样可以加快目录共享的速度。子树检查是一种安全特性，用于检查共享目录的父目录和祖先目录的权限，但在某些情况下可能会影响性能。使用 (no_subtree_check) 参数可以禁用这种检查。

测试

首先在共享目录 /share 下创建一个文件：

$ echo 123 > /share/123.txt

在另一台主机上，挂载该共享目录：

$ sudo mount -t nfs 192.168.223.129:/share /mnt

其中，192.168.223.129 是 NFS Server 的 IP，/share 是该 server 的共享目录，/mnt 是客户端要挂载到本机的目的目录。

执行完成后如果没有任何错误信息则代表成功，查看当前主机的 /mnt 目录：

$ ls /mnt
123.txt

$ cat /mnt/123.txt
123

发现之前在 NFS Server 的 /share 里创建的文件，出现在了客户端主机的 /mnt 目录下，代表 NFS Server 搭建成功。

StorageClass

# nfs-storage.yaml
# kubectl apply -f nfs-storage.yaml
apiVersion: storage.k8s.io/v1
kind: StorageClass
metadata:
  name: nfs-storage
# 使用上面的自定义 provisioner 名称
provisioner: nfs-provisioner
parameters:
  # archiveOnDelete 指定为 false 表示删除 PVC 后 PV 也会被删除。
  # 如果想在 PVC 被删除后仍旧保留数据的，可指定为 true
  archiveOnDelete: "false"

验证：

创建 pvc

kind: PersistentVolumeClaim
apiVersion: v1
metadata:
  name: test-claim
  annotations:
    # 与 nfs-StorageClass.yaml metadata.name 保持一致
    volume.beta.kubernetes.io/storage-class: "nfs-storage"
spec:
  accessModes:
    - ReadWriteMany
  resources:
    requests:
      storage: 1Mi

查看是否 Bound：

$ k get pvc
NAME           STATUS    VOLUME                                     CAPACITY   ACCESS MODES   STORAGECLASS   AGE
test-claim     Bound     pvc-969d9309-4510-46ac-90c1-8e9cc1d8f2ec   1Mi        RWX            nfs-storage    22m

设置为默认 storageclass

$ kubectl patch storageclass <storageclass-name> -p '{"metadata": {"annotations":{"storageclass.kubernetes.io/is-default-class":"true"}}}'

比如现在有一个名为 nfs-storage 的 sc，要将其设置为默认 sc：

$ k get sc
NAME          PROVISIONER       RECLAIMPOLICY   VOLUMEBINDINGMODE   ALLOWVOLUMEEXPANSION   AGE
nfs-storage   nfs-provisioner   Delete          Immediate           false                  41m

$ kubectl patch storageclass nfs-storage -p '{"metadata": {"annotations":{"storageclass.kubernetes.io/is-default-class":"true"}}}'

执行后，发现名字后面多了一个 (default)

$ k get sc
NAME                    PROVISIONER       RECLAIMPOLICY   VOLUMEBINDINGMODE   ALLOWVOLUMEEXPANSION   AGE
nfs-storage (default)   nfs-provisioner   Delete          Immediate           false                  46m
