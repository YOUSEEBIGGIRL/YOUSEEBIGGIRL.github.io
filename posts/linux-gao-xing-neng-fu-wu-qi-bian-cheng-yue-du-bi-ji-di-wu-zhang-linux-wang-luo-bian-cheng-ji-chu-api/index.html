1. 字节序

字节序：内存中存储多个字节的方式。

例如一个 16 进制数，它由两个字节组成，内存中存储这两个字节有两种方式，分别为大端字节序和小端 字节序。

大端：高位字节存放在低地址，低位字节存放在高地址。

小端：高位字节存放在高地址，低位字节存放在低地址。

如何区分字节的高低位和地址的高低位？

0x12 0x34 高位字节 低位字节 ← 字节增长方向 低地址 高地址 → 内存增长方向

1.1 网络字节序和主机字节序

主机字节序：某个给定系统所用的字节序称为主机字节序。不同的操作系统采用的字节序不同，比如 macos 采用的是大端，而 linux 采用的是小端。

网络字节序：为了在信息传输时，屏蔽掉不同硬件结构上的字节序的差异，TCP/IP协议规定，所有在网络上传输的多字节整数都以大端序编码，所以大端序就是网络字节序。

字节序转换的函数：

#include <netinet/in.h>

// 主机字节序转网络字节序，两个函数的区别在于长度不同
// 较短的函数可以用来转换端口号，较长的可以转换 ip 地址
uint16_t htons(uint16_t host16bitvalue);
uint32_t htonl(uint32_t host32bitvalue);

// 网络字节序转主机字节序
uint16_t ntonhs(uint16_t net16bitvalue);
uint32_t ntonhs(uint32_t net32bitvalue);

2. socket api

2.1 创建 socket

#include <sys/socket.h>

int socket(int family, int type, int protocol);

// 返回：若成功则为非负描述符，出错则为 -1

参数描述：

  1. family 用来选择通信协议，参数主要有以下一些常用类型

    • AF_UNIX, AF_LOCAL：用于本地通信，Unix Domain Socket

    • AF_INET：用于 IP4

    • AF_INET6：用于 IP6

  2. type 指明套接字类型

    • SOCK_STREAM 字节流套接字，即 tcp

    • SOCK_DGRAM 数据包套接字，即 udp

    • SOCK_NONBLOCK 将 socket 设置为非阻塞
    • SOCK_CLOEXEC fork 子进程中关闭该 socket

  3. protocol 在前两个参数构成的协议集合下，再选择一个具体的协议，通常设置为 0 即可（代表默认协议）

2.2 命名 socket

socket 命名：将一个 socket 和 socket 地址绑定。

#include <sys/socket.h>

int bind(int sockfd, const struct *myaddr, socklen_t addrlen);

// 成功返回 0，出错返回 -1

使用：

struct sockaddr_in servaddr;

bzero(&servaddr, sizeof(servaddr));	// 初始化结构体

servaddr.sin_family = AF_INEF; // ip4
servaddr.sin_addr.s_addr = htonl(INADDR_ANY); // 指定 ip，并转换为网络序
servaddr.port = htons(13);	// 指定端口号，并转换为网络序

// 第二个参数需要强转成 struct sockaddr，sockaddr 是一个通用的套接字地址结构
bind(&servaddr, (struct sockaddr*) &servaddr, sizeof(servaddr));

2.3 监听 socket

#include <sys/socket.h>

int listen(int sockfd, int backlog);

// 成功返回 0，出错返回 -1
