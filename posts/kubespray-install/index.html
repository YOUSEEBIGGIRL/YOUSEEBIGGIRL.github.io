通过 Kubespray 搭建 k8s 集群

2023-04-16
2分钟阅读时长

⚠️ 安装前最好确保你的主机够干净，我的虚拟机就因为之前自行用二进制搭建过，导致使用 kubespray 安装出现各种奇奇怪怪的错误，最后删除重新创建了一台，一次就安装成功了

初始化

apt install pip

# ssh 登录要用，不然报错 "to use the 'ssh' connection type with passwords or pkcs11_provider, you must install the sshpass program
apt install sshpass

pip install netaddr # 不确定这玩意的必要性，最好还是装了吧

# 不要手动安装下面这些东西，手动安装的 ansible 可能与 Kubespray 要求的版本不匹配
#pip install --upgrade jinja2
#apt install ansible

# 不要 clone 这个，可能有问题
#git clone https://github.com/kubernetes-incubator/kubespray.git

# 从 release 里下载
wget https://github.com/kubernetes-sigs/kubespray/archive/refs/tags/v2.21.0.tar.gz
# 解压上面下载下来的压缩包
tar -C . -xvf v2.21.0.tar.gz

cd kubespray-2.21.0

# 安装符合版本的依赖，这里会安装 ansible
pip install -U -r requirements.txt

修改配置

cd kubespray-2.21.0
# 复制一份配置进行修改
cp -rfp inventory/sample inventory/k8s
vim inventory/k8s/inventory.ini

配置如下，保证 IP 正确，其中 master 节点要指定 etcd_member_name，worker 节点该值置为空

同理，[kube_control_plane] 和 [etcd] 里写 master 节点，[kube_node] 里写 worker 节点

# ## Configure 'ip' variable to bind kubernetes services on a
# ## different ip than the default iface
[all]
k8s-master ansible_host=192.168.223.128 ip=192.168.223.128 ansible_ssh_user=root ansible_ssh_pass=root etcd_member_name=k8s-master
k8s-worker1 ansible_host=192.168.223.129 ip=192.168.223.129 etcd_member_name=""
k8s-worker2 ansible_host=192.168.223.130 ip=192.168.223.130 etcd_member_name=""
# node1 ansible_host=95.54.0.12  # ip=10.3.0.1 etcd_member_name=etcd1
# node2 ansible_host=95.54.0.13  # ip=10.3.0.2 etcd_member_name=etcd2
# node3 ansible_host=95.54.0.14  # ip=10.3.0.3 etcd_member_name=etcd3
# node4 ansible_host=95.54.0.15  # ip=10.3.0.4 etcd_member_name=etcd4
# node5 ansible_host=95.54.0.16  # ip=10.3.0.5 etcd_member_name=etcd5
# node6 ansible_host=95.54.0.17  # ip=10.3.0.6 etcd_member_name=etcd6

# ## configure a bastion host if your nodes are not directly reachable
# [bastion]
# bastion ansible_host=x.x.x.x ansible_user=some_user

[kube_control_plane]
k8s-master

[calico_rr]

kube_control_plane
kube_node
calico_rr

开始部署

执行下面的命令进行部署，中间可能需要等待 10 分钟左右。

ansible-playbook -i inventory/k8s/inventory.ini cluster.yml -b -vvv

最终如果执行结果如下，即所有节点 failed=0，代表安装成功

错误记录

1

报错 Ansible must be between 2.11.0 and 2.13.0 exclusive

执行 python3 -m pip install –upgrade –user ansible 后还是报错，发现是版本又太高了：

ansible –version ansible [core 2.14.4]

2

报错 the output has been hidden due to the fact that ’no_log: true’ was specified for this result

vim inventory/k8s/group_vars/all/all.yml

修改下面这行为 true

unsafe_show_logs: true

3

报错：

The checksum for /tmp/releases/calico-v3.25.1-kdd-crds/v3.25.1.tar.gz did not match 361b0e0e6d64156f0e1b2fbfd18d13217d188eee614eec5de6b05ac0deaab372; it was 4d6b6653499f24f80a85a0a7dac28d9571cabfa25356b08f3b438fd97e322e2d.

尝试手动下载

cd /tmp/releases/calico-v3.25.1-kdd-crds

wget https://github.com/projectcalico/calico/archive/v3.25.1.tar.gz

不行，执行部署命令会清空该目录

解决：

不要 clone 下面这个 master 分支，可能有问题

git clone https://github.com/kubernetes-incubator/kubespray.git

从 release 里下载

wget https://github.com/kubernetes-sigs/kubespray/archive/refs/tags/v2.21.0.tar.gz

后安装就没有这个问题了
